CI/CD Pipeline Demo - Java Maven with AKS Deployment

This project demonstrates a complete CI/CD implementation for a Java Maven application. The pipeline automates everything — from build, test, code quality, artifact management, Dockerization, and deployment to Azure Kubernetes Service.

Created by Vivek, working at Cognizant Technology Solutions.

Explore Full Workflow

GitHub Actions acts as the CI/CD engine. It triggers the pipeline on every code push or pull request, running all stages sequentially.

Maven compiles, tests, and packages the Java application into a JAR file. The build ensures all dependencies and unit tests pass before moving forward.

SonarQube performs static code analysis to check code quality, maintainability, and security vulnerabilities before deployment.

JFrog Artifactory securely stores versioned build artifacts (JARs). This allows us to trace and pull the exact build we want for deployment.

Docker packages the JAR into a container image, ensuring consistent runtime across all environments. The image is pushed to Docker Hub.

Azure Kubernetes Service (AKS) pulls the Docker image and deploys it as a pod behind a LoadBalancer service, making the app accessible with scalability and reliability.
